# 业务需求分析

## 销售子系统

### 销售人员

#### 草拟订单
销售人员输入新的订单草稿数据，系统接收订单数据后创建订单草稿，并使该订单处于“草稿”状态。

#### 提交评审
销售人员可以按条件（暂时忽略）查询可提交评审的订单草稿列表，从中选取需提交评审的订单草稿，
系统按各类评审要求处理所选订单，形成评审材料推送给质量管理、运输等子系统开展评审，提交评审
的订单将处于“评审”状态。
